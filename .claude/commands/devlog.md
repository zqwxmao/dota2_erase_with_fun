你是 Dota 2 英雄消消乐项目的开发日志管理员。根据本次会话中完成的工作，生成或更新开发日志。

## 执行步骤

### 1. 收集本次操作信息
回顾当前会话中所有已完成的操作，提取：
- 具体做了什么（新增/修改/删除/修复/讨论/测试）
- 涉及哪些文件
- 属于哪个角色维度（产品/研发/测试）
- 操作结果和影响

### 2. 写入日报
文件路径: `devlog/daily/YYYY-MM-DD.md`（使用当天日期）

如果当天日报已存在则**追加**，不存在则创建。格式如下：

```markdown
# 开发日报 YYYY-MM-DD

## [HH:MM] 操作标题
- **角色**: 产品 / 研发 / 测试
- **类型**: 新增功能 / 修改功能 / Bug修复 / 配置调整 / 方案讨论 / 测试验证
- **内容**: 具体描述做了什么
- **涉及文件**:
  - `path/to/file1`
  - `path/to/file2`
- **结果**: 操作结果和当前状态
```

### 3. 检查是否需要更新周报/月报
- 如果是本周第一条日报，创建周报模板 `devlog/weekly/YYYY-WXX.md`
- 如果是本月第一条日报，创建月报模板 `devlog/monthly/YYYY-MM.md`

周报格式：
```markdown
# 开发周报 YYYY 第XX周 (MM-DD ~ MM-DD)

## 产品进展
- （汇总本周产品相关工作）

## 研发进展
- （汇总本周研发相关工作）

## 测试进展
- （汇总本周测试相关工作）

## 本周数据
- 新增功能: X 项
- 修复 Bug: X 个
- 代码变更文件: X 个

## 下周计划
- （待填写）
```

月报格式：
```markdown
# 开发月报 YYYY年MM月

## 本月总览
（一句话总结本月进展）

## 产品维度
### 已完成需求
### 进行中需求
### 待排期需求

## 研发维度
### 已完成开发
### 技术债务
### 架构变更

## 测试维度
### 测试覆盖情况
### 遗留 Bug
### 质量评估

## 本月数据
- 新增功能: X 项
- 修复 Bug: X 个
- 代码变更文件: X 个

## 下月计划
- （待填写）
```

### 4. 同步更新对应的角色计划文件
如果本次操作涉及某个角色维度的计划变更，同步更新 `devplan/` 下对应文件的状态。

### 5. 输出确认
完成后输出简要确认：
- 日报已更新/创建
- 周报/月报是否有变动
- 涉及的角色维度
